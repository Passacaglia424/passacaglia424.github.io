    ————序

    注：由于学习时使用的是Windows 10系统，所以以下内容仅保证适用于windows系统，Linux系统下可能有微小差别(如OsString部分)。语法有点像特化的C++，有自己很多独特机制。Rust虽然官方教程友好，但是不适合作为入门语言。

    学习参考视频：https://www.bilibili.com/video/BV1hp4y1k7SV?p=36

    文件命名规范：文件名若有单词，中间加下划线，如hello_world.rs。
    常量命名规范：全大写字母。
    函数名名规范：小写字母，单词间下划线分开。Rust函数可定义在main函数后。

    Rust的main函数没有参数，没有返回。fn main{}。

    Rust marco(宏)，会在宏名称后面加’!’。Rust的缩进是四个空格，而不是tab键。

    Rust是“ahead of time”(预先编译)的语言，编译成功直接生成二进制文件，在windows下还会额外生成一个.pdb的调试文件。

    项目创建

    cargo有官方中文文档。
    使用包管理器cargo创建(安装Rust时自带，无需额外安装)。
    命令: cargo new <项目名> 在当前路径下生成项目。

    项目内可能存在的文件含义：
    Cargo.toml 项目配置文件
    .gitignore 用于git
    Cargo.lock 关于依赖项的信息。

    可在项目下使用的其他命令

    cargo build 构建项目
    cargo run 编译运行代码
    cargo check 只检查代码能通过编译，但不生成可执行文件。
    cargo update 按照Cargo.lock更新依赖
    cargo fmt 代码格式化

    按层级划分：

    Package

    包。有Cargo的特性，测试、共享、构建crate。
    包含一个Cargo.toml,描述如何构建crate。只能包含0-1个library crate，但可以包含任意个binary crate。binary crate放在src/bin目录下。
    Cargo.toml可添加引用外部包。std属于外部包但不用添加引用，但是使用时还需要use。

    Crate

    单元包。一个模块树，可产生一个library或可执行文件。
    分为binary crate和library crate。
    src/main.rs:binary crate的crate root。Rust编译器从这里开始组成crate的根module。
    src/lib.rs:library crate的crate root。一般将自己写的库或是公用业务逻辑放在这里。

    Module

    模块，定义在文件中。使用mod关键词创建，可嵌套。
    模块可以定义私有边界，Rust中所有变量，结构体、模块、函数等都默认是私有的。父模块无法访问子模块条目，但子模块可以访问祖先的。
    如果模块名后跟”;”而非代码块，那么指在此文件名中定义了模块。如在文件A.rs中定义了mod B;那么实际在src/A/B.rs。(此A为目录名)

    Path

    为了在模块中找到某个条目而使用。
    -绝对路径：从crate root开始，使用crate名或字面值crate。
    -相对路径：当前模块开始，使用self、super或当前模块标识符。

    use的使用

    pub use
    嵌套路径：一次引用多个来自相同的部分:

    use std::io::{self,write}

    self表示自身，另外也可使用通配符*。

    1.通用的编程概念

    隐藏(shadowing)

    使用let可以给之前的同名变量声明新的变量，新的变量会隐藏之前的同名变量。
    如:

    let x=5;
let x=x+1;

    Rust不允许变量使用时未初始化。

    1.1数据类型

    1.1.1标量数据类型

    u32:32位的无符号整型。范围-(2^n-1)~2^n-1。
    i32:32位的有符号整型。范围0-2^n-1，为默认类型。
    u/i8、u/i16,u/i64,u/i128同理，数字为位数大小。
    isize/usize:由计算机架构决定大小，如64位系统就是64。
    f32:32位浮点数类型。
    f64:64位浮点数类型，为默认类型。
    bool:值为true/false。
    char:4字节，使用unicode标量值。范围U+0000-U+D7FF,U+E000-U+10FFFF

    Rust不支持自动类型转换，必须显式转换。

    1.1.2复合数据类型

    Tuple:元组，圆括号声明，具有固定长度。类似python。

     let tup: (i32, f64, u8) = (500, 6.4, 1);//声明一个三元素元组。

    ():单元类型，仅有一个值()，不需要内存，相当于0长度元组。函数没有返回值，那么返回一个 ()。

    数组：分配在堆栈上。类似js。rust不允许访问超出索引的内存。

    let a = [1, 2, 3, 4, 5];
let a:[i32;5] = [1, 2, 3, 4, 5];
let a = [3;5];//分配了5个元素，都是3。

    结构体

    struct Rectangle{
    width:u32;
    height:u32;
}
let rect= Rectangle{
    width:32,
    height:40,
};//赋值
/*当仅email字段user1与user2不同时，可这样初始化*/
 let user2 = User {
    email: String::from("another@example.com"),
    ..user1
};
/*赋值的非"Copy特征"内容所有权会转移，但其他部分还能使用*/

    结构体实例声明为可变的，才能修改其中的字段，Rust 不支持将某个结构体某个字段标记为可变。

    enum枚举，值只能是其中之一。
    结构体、枚举类型中可定义方法。访问成员 枚举名::成员名

    impl Rectangle{
    fn area(&self)->u32{
        self.width=self.length;
    }
}

    rust中没有->运算符，会自动引用和解引用。

    1.1.3集合数据类型

    标准库定义的数据类型，储存在堆上。

    vector

    写法Vec。创建方式有：

    let v:Vec<i32>=Vec::new();
Vec::with_capacity(capacity)//创建指定容量
let vec![1,2,3];
/*与枚举的结合，实现储存不同数据*/
enum Cell{
    Int(i32);
    Float(f64);
    Text(String);
}
let v=vec![Cell::Int(3),Text(String::from("hello))];

    操作：

    v.push(4);//添加元素。
v.get(2);//获得引用。或者使用下标索引也是引用。
    String

    这是来自标准库的类型，除此之外还有&str、OsString、CString、CStr等类似的字符串类型。Rust不允许对String进行索引，但允许按照字形簇的索引边界进行切割。String类型使用UTF-8编码。底层的数据存储格式实际上是u8。

    let mut s=String::from("foo");
s.push_str("bar");//添加字符串到之后，该方法不会获得所有权。
s.push('s');//添加单个字符
let s=format!("{}-{}",s,x);//用宏拼接s和x字符串，不会获得所有权。
for (k,v) in &s{//遍历
    println!("{}{}",k,v);
}
//split_whitespace方法：按空格分割字符串

    +号也可以连接字符串，不能两个字符串直接相加，需要其中一个是引用形式，会将其转化为字符串切片。不引用的会失效。
    字符串切片：

    被切片引用的字符串禁止修改值。
    字形簇：真正接近人类语言的表示，一个个真正的字符。

    HashMap

    哈希表，用键值对的方式储存HashMap<K,V>。不是预导入类型。K,V分别必须是同一类型。
    预导入(preclude)：不需要额外use，直接就可使用。haspMap不是，需要use std::collections::HashMap;

    let mut hp=HashMap::new();
hp.insert(3，5);//编译器可以推断出K,V的类型了。
/*通过元素类型为元组的vector加collect()方法来生成HashMap。*/
let v=vec![1,2];
let w=vec![3,4];
/*将别的类型转为迭代器再直接导入哈希表*/
let s:HashMap<_,_>=v.iter().zip(w.iter()).collect();
s.entry(30).or_insert(88);//如果没有K=30的值，插入30,88，否则不插入
/*or_insert()返回值:K存在返回对应V的可变引用；不存在就插进去再返回这个可变引用。*/

    1.2控制流

    1.2.1控制语句

    判断if/else if/else,判断表达式不加括号。
    可用于作为变量条件声明，但是要求返回类型相等。

    let number = if true {5} else {6};

    无限循环loop，break退出。条件循环while，判断表达式也不加括号。

    Rust通过break退出循环可以返回值，如break i;将i返回到循环外面，可以通过左值接收。

    范围循环for … in … 类似python。设置1..4可表示1-3的范围，1..=4表示1-4,’a’..=’z’表示a-z,rev()函数反向。
    模式匹配match,举例(绑定值的模式匹配):

    match Coin{
    Coin:dollar=>1,
    Coin:yuan=>4,
    Coin:Quarter=>(state){
        println!{"Hello {:?}",state};
        25
    }
}

    通配符为其余情况,()表示什么都没发生。rust强制要求处理所有情况，可用=>()

    1.2.2错误处理

    不可恢复错误

    执行panic!宏，默认展开清理调用栈，或选择不清理直接退出程序:在cargo.toml中设置panic=’abort’。
    cargo build/run中不带–release参数会带有调试信息。

    可恢复的错误

    Result枚举类来处理错误，这类错误类似java里的异常。可通过结合match表达式处理错误，也可以用闭包。

    enum Result<T, E> {
    Ok(T),//操作成功Ok变体返回T类型
    Err(E),//操作失败Ok变体返回E类型
}

    .unwrap()方法：match的快捷方法，如果Ok返回值，如果Err调用panic!。
    .expect()方法可以自定义错误信息。

    传播错误

    比如函数返回值为Result<String,io:Error>，将错误抛回调用者。
    ?运算符：在返回值为Result的函数后面加?，也能起到返回错误的作用。实际上隐式调用了from函数。

    1.3格式化输出

    print! 将格式化文本输出到标准输出，不带换行符
    println! 同上，但是在行的末尾添加换行符
    format! 将格式化文本输出到 String 字符串
    eprint!，eprintln!标准错误输出

    占位符

    {}:常规占位符。
    {:?}:适用于调试场景。如声明#[derive(Debug)]后可输出结构体。
    {:#?}：格式更清晰的{:?}
    {:.2}:输出小数点后两位

    泛型

    泛型实际上是含有占位符的模板，参考C++。编译时将占位符换成具体的类型参数。
    如:

    fn largest<T>(list:&T)->T{...} //函数中的泛型

    2.Rust核心机制

    Rust的核心机制。储存在stack上的数据必须有固定的大小，也就是copy trait的，编译时大小未知或运行时大小变化的数据存在heap上。
    元组内的都是copy trait的，那么它本身也是。

    2.1所有权机制☆

    2.1.1引用

    引用：允许你使用某些值而不取得其所有权，因为Rust中变量一旦离开其所有权，就会失效(主要用于阻止二次内存释放漏洞)。用&表示。除此之外，直接赋值或调用.move()，会导致所有权转移，使用.clone(),相当于生成一个新的同样值的对象。.get()可以获得引用。默认不允许修改引用的值。
    可变引用：&mnt [变量] 方式,函数中可变引用作参数，实参也需要可变。
    引用的作用域：从声明到最后一次使用。
    Rust中，编译器可以保证引用永远不会是悬空引用，即不会指向的内容已提前被释放。
    任意时刻只能满足之一：①一个可变引用 ②任意个不可变引用。也就是说，一个变量在其作用域内不能同时有可变和不可变的引用。
    pub关键字：公有的。对enum设置pub,里面都是pub的。而struct是pub，里面不一定是pub,mod也是。
    切片slice不会改变所有权。

    引用是是引用，需解引用使用，不能比较整数和引用。

    2.1.2函数中所有权变化

    如果是基本数据类型，进入函数时不失去所有权；堆中的变量进入函数会失去所有权，因此若需要保留，则应该引用传参。
    被当作函数返回值的变量所有权将会被移动出函数并返回到调用函数的地方，而不会直接被无效释放。

    2.3Trait

    把方法签名放在一起。来定义实现某种目的所必须的一种行为。类似于接口。译名为“特征”。
    只能在本地crate实现trait。如果想在其他crate中实现trait，需要use引入并设置成公有。但是必须在本地作用域，父类型中不可见。

    pub trait Summary {
    fn summarize(&self) -> String {//实现默认方法
        String::from("(Read more...)")
    }
    fn read(&self)->String;//不实现默认方法
}
impl Summary for NewsArticle {//为NewArticle实现方法。
    fn summarize(&self) -> String {
        format!("{}, by {} ({})", self.headline, self.author, self.location)
    }
}
/*Trait作为参数*/
pub fn notify(item: impl Summary) {
    println!("Breaking news! {}", item.summarize());
}
/*Trait Bound形式语法糖*/
pub fn notify<T: Summary>(item: T) {
    println!("Breaking news! {}", item.summarize());
}

    默认实现的Trait

    该类Trait可以通过几种形式给予声明的结构、函数泛型参数等使用。
    如以下派生宏方式为结构体Earth实现Debug Trait:

    #[derive(Debug)]
struct Earth{
    water:i32;
    soil:f32;
};

    常见Trait的用途：
    Debug Trait:便于调试，如结构体实现后使用调试打印会打印更清晰的结构
    Copy Trait:可以简单地复制类型变量，但不复制指针指向的内容。不转移所有权。
    Clone Trait:Copy是Clone的子特征，实现了Copy也要实现Clone。
    Drop Trait:实现了该Trait就不能实现Clone Trait。实现后会在作用域结束后自动释放资源。
    PartialEq Trait:作用于泛型，允许两个该泛型进行相等比较。
    PartialOrd Trait:作用于泛型，允许两个该泛型进行不等比较。

    该部分可参照:https://zhuanlan.zhihu.com/p/369264606

    remove() 删除并返回字符串中指定位置的字符,参数为索引。
    truncate() 删除字符串中从指定位置开始到结尾的全部字符
